Assignment 2: Threads

Deadline: 25th September, 2012 (IST)

A penalty of 20% per day would be imposed for late submission.

This assignment is to be implemented in three person teams.

Objective

Part 1

Read, understand and summarize the semantics for POSIX threads (pthreads). A good place to begin is here.

Part 2

You are required to implement a subset of the POSIX interface to pthreads in Pintos. The following functions (with the exact prototype as provided in the POSIX interface) must be implemented:


pthread_create (pthread_t *thread, const pthread_attr_t *attr, void *(*start_routine) (void *), void *arg);

pthread_join(pthread_t thread, void **retval);

pthread_exit (void *status);

pthread_cancel (pthread_t thread);

pthread_attr_init (pthread_attr_t *attr);

pthread_attr_destroy (pthread_attr_t *attr);

pthread_setschedparam (pthread_t thread, int policy, const struct sched_param *param);


The following thread attributes must be implemented: detachstate, inheritsched, schedpolicy. The following scheduling policies must be implemented: SCHED_FCFS, SCHED_RR, SCHED_PRIORITY. Note that the above functions must be implemented as system calls in Pintos.

Part 3

Use your (above) implementation and your implementation of message queues (in assignment #1) to solve the Producer-Consumer problem (discussed in class).
