Objective

The file system provided in Pintos, among others, has the following deficiencies:
  • Fixed File Sizes: File sizes are fixed at creation-time, that cannot be changed thereafter
  • No subdirectories: Pintos has a flat filesystem, not allowing the creation of subdirectories.
  • Files need to be stored contiguously, that can lead to external fragmentation


    • All the above definiciencies must be addressed to:
  • Implement an indexed filesystem so that files can grow; this will also address the problem of external fragmentation. Provide for direct, indirect and doubly indirect blocks.
  • Allow for hierarchical subdirectories; this will need the implementation of the following system-calls:
    int mkdir(const char *pathname, mode_t mode);
    int readdir(unsigned int fd, struct old_linux_dirent *dirp, unsigned int count);
    int chdir(const char *path);
    See the Unix man-pages for details on these calls.
  • Provide for a buffer cache: the buffer cache must be checked before visiting the disk.
